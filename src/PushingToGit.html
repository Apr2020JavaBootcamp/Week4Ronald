Pushing your projects to Github

Pushing to Github is one of those things you can say you definitely need to learn, particularly when it involves a group project, which is likely the case almost anywhere you work at. This goes for Github in general, but for the purposes of this post, I will only cover pushing projects to Github.

  1. Create an account on Github
  2. Take your Github ID and associated email address and open your IDE
  3. Create a new project. Open the terminal and at the bottom left of the screen and enter the following (excluding the quotations) if you have never pushed before:
    • git config --global user.name "your Github ID"
    • git config --global user.email "your email associated to your Github ID"
    • git config -l
      ^this one displays a list that should contain the information you just entered into the system
  4. Create a README file that will give a general description of the project as follows:
    • echo "The text that will be in the file" >> README.md
  5. Now it's time to populate your local repository with your current project:
    • git init
    • git add .
    • git commit -m ""Text describing the project you're pushing""
  6. Now go to Github website and at the top right of the screen, click on the '+' sign and select 'New Repository'.
  7. Give the repository a name and leave the rest settings as their default. Copy the repository URL.
  8. In your terminal enter the following:
    • git remote add origin "paste the URL you just copied"
    • git push origin master
  9. Wait a few seconds then refresh your repository on Github to see your project there.
